# レビュー記録の置き場

## 目的

設計レビュー、ソースレビュー、テストレビューを `docs` 配下で一箇所に管理する。

## 配置ルール

- レビュー記録は `docs/レビュー/{対象}/` に置く。
- `{対象}` は `サイト` `タイピング` `ブロック崩し` など対象単位で分ける。
- レビュー結果を反映した正式な要件定義書・設計書は、元の配置先で更新する。

## 命名規則

| 種別 | 命名パターン | 例 |
|------|--------------|----|
| ゲーム基本設計レビュー | `ゲーム基本設計レビュー_{対象}.md` | `ゲーム基本設計レビュー_ブロック崩し.md` |
| ゲーム詳細設計レビュー | `ゲーム詳細設計レビュー_{対象}.md` | `ゲーム詳細設計レビュー_ブロック崩し.md` |
| ソースレビュー | `ソースレビュー_{対象}.md` | `ソースレビュー_ブロック崩し.md` |
| テストレビュー | `テストレビュー_{対象}.md` | `テストレビュー_ブロック崩し.md` |

## パス例

- `docs/レビュー/ブロック崩し/ゲーム基本設計レビュー_ブロック崩し.md`
- `docs/レビュー/ブロック崩し/ゲーム詳細設計レビュー_ブロック崩し.md`
- `docs/レビュー/ブロック崩し/ソースレビュー_ブロック崩し.md`
- `docs/レビュー/ブロック崩し/テストレビュー_ブロック崩し.md`

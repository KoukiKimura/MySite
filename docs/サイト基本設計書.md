## 2. ルーティング設計

### 2.1 ルーティング方式
- **Hash ベースルーティング**（`#/path`）を採用
- サーバー設定不要で静的サイト（ConoHa Wing）と相性が良い
- 単一HTMLファイル（SPA的構成）でページ遷移を管理

### 2.2 ルート定義

| ルート | 画面 | モジュール | 状態 |
|--------|------|-----------|------|
| `#/` | トップページ | `portal/pages/home.js` | 実装 |
| `#/games/typing` | タイピングゲーム説明 | `games/typing/pages/description.js` | 実装 |
| `#/games/typing/play` | タイピングゲーム画面 | `games/typing/pages/play.js` | 実装 |
| `#/ranking` | ランキング | `user/pages/ranking.js` | モック |
| `#/login` | ログイン | `user/pages/login.js` | モック |
| `#/about` | About | `portal/pages/about.js` | 実装 |
| `#/contact` | お問い合わせ | `portal/pages/contact.js` | 実装 |
| （上記以外） | 404 | `portal/pages/not-found.js` | 実装 |

### 2.3 ルーター設計

```javascript
// shared/router/router.js
// Hashベースの軽量ルーター

const routes = {
  '/':                    () => import('../../portal/pages/home.js'),
  '/games/typing':        () => import('../../games/typing/pages/description.js'),
  '/games/typing/play':   () => import('../../games/typing/pages/play.js'),
  '/ranking':             () => import('../../user/pages/ranking.js'),
  '/login':               () => import('../../user/pages/login.js'),
  '/about':               () => import('../../portal/pages/about.js'),
  '/contact':             () => import('../../portal/pages/contact.js'),
};

// 未定義ルートへのアクセス時は404ページを表示
const fallback = () => import('../../portal/pages/not-found.js');
```

#### ローディング表示

ページ遷移時に動的importが完了するまでの間、メインコンテンツ領域にローディングインジケーター（CSSスピナー）を表示する。

## 7. 状態管理設計

### 7.1 方針
- グローバルな状態管理ライブラリは使用しない
- **`CustomEvent`** によるイベント駆動で画面間連携
- ページ固有の状態は各ページモジュール内で管理

### 7.2 グローバル状態

| 状態 | 保存先 | 用途 |
|------|--------|------|
| ログイン状態 | `sessionStorage` | モック認証の状態管理 |
| 現在のルート | メモリ（Router内） | ページ表示切り替え |

### 7.3 イベント設計

| イベント名 | 発火元 | リスナー | データ |
|-----------|--------|---------|--------|
| `route-change` | Router | PageContainer | `{ path: string }` |

## 10. セキュリティ設計

### 10.1 フロントエンド対策

| 対策 | 内容 |
|------|------|
| XSS防止 | DOM操作時に `textContent` を使用、`innerHTML` の使用を制限 |
| CSP | `Content-Security-Policy` ヘッダーでスクリプト・スタイル読み込みを制限 |
| 外部リソース | Google Fonts等はSRI（Subresource Integrity）で検証 |
| Google Forms | iframe `sandbox` 属性で権限制限 |

### 10.2 CSPポリシー（案）

```
Content-Security-Policy:
  default-src 'self';
  script-src 'self';
  style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
  font-src 'self' https://fonts.gstatic.com;
  img-src 'self' data:;
  frame-src https://docs.google.com;
```

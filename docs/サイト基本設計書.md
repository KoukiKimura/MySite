### 1.2 サイトモード切替方針

| 項目 | 基本方針 |
|------|----------|
| モード種別 | `キッズ` / `ティーンズβ` |
| 切替UI配置 | ヘッダー常設、モバイルメニュー内にも配置 |
| 適用範囲 | トップ、カテゴリ、検索体験、特集、キーワード、UIトーン |
| 共通ページ | ゲーム説明、ゲーム画面、ランキング、ログイン、About、お問い合わせ、各種ガイドページ |
| 切替時遷移 | 現在位置に関わらず対象モードのトップ `#/kids` または `#/teens` へ遷移 |
| 初期表示 | `localStorage` の最終モードを優先、未保存時は `キッズ` |
| 将来方針 | 外部検索、安全フィルタ、推薦ロジックもモードごとに分岐 |

### 1.3 主要ユーザーフロー

1. ユーザーが `#/` にアクセスする
2. ルーターが保存済みモードを参照し、`#/kids` または `#/teens` に遷移する
3. トップページで検索、人気キーワード、カテゴリ導線、特集カードから次の導線を選ぶ
4. カテゴリページではカテゴリ内のみ検索し、カードや一覧からゲームや案内ページへ移動する
5. ゲームをプレイした結果は端末内履歴に保存され、ランキングとおすすめ特集に反映される
6. ログイン済みユーザーは履歴確認、ランキング確認、管理画面モックへの遷移ができる

## 2. ルーティング設計

### 2.1 ルーティング方式
- Hash ベースルーティング（`#/path`）を採用する
- 静的ホスティング（ConoHa Wing）に合わせてサーバーリライト不要で運用する
- トップとカテゴリにモード別ルートを導入するため、静的ルートに加えてパターンマッチング型のルーターへ拡張する

### 2.3 ルーター設計

```javascript
// shared/router/router.js

const ROUTES = [
  {
    pattern: /^\/$/,
    load: () => import('../../portal/pages/entry.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/(kids|teens)$/,
    load: () => import('../../portal/pages/home.js'),
    getParams: ([, mode]) => ({ mode }),
  },
  {
    pattern: /^\/(kids|teens)\/([a-z0-9-]+)$/,
    load: () => import('../../portal/pages/category.js'),
    getParams: ([, mode, category]) => ({ mode, category }),
  },
  {
    pattern: /^\/games\/typing$/,
    load: () => import('../../games/typing/pages/description.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/games\/typing\/play$/,
    load: () => import('../../games/typing/pages/play.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/ranking$/,
    load: () => import('../../user/pages/ranking.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/login$/,
    load: () => import('../../user/pages/login.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/admin$/,
    load: () => import('../../admin/pages/dashboard.js'),
    getParams: () => ({}),
    guard: 'mock-auth',
  },
  {
    pattern: /^\/parents$/,
    load: () => import('../../portal/pages/parents.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/guide$/,
    load: () => import('../../portal/pages/guide.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/safety\/links$/,
    load: () => import('../../portal/pages/safety-links.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/policy\/content$/,
    load: () => import('../../portal/pages/content-policy.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/policy\/privacy$/,
    load: () => import('../../portal/pages/privacy.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/about$/,
    load: () => import('../../portal/pages/about.js'),
    getParams: () => ({}),
  },
  {
    pattern: /^\/contact$/,
    load: () => import('../../portal/pages/contact.js'),
    getParams: () => ({}),
  },
];

function resolveRoute(path) {
  for (const route of ROUTES) {
    const matched = path.match(route.pattern);
    if (matched) {
      return {
        loader: route.load,
        params: route.getParams(matched),
        guard: route.guard ?? null,
      };
    }
  }

  return {
    loader: () => import('../../portal/pages/not-found.js'),
    params: {},
    guard: null,
  };
}
```

#### ルーティング補足
- `#/kids/:category` と `#/teens/:category` はカテゴリマスタに存在する値のみ許可し、不正なカテゴリは 404 とする
- `#/admin` はモック認証済みユーザーを前提とし、未ログイン時は `#/login` へ誘導する
- ヘッダーのサイト切替は `setSiteMode(mode)` 実行後に `#/kids` または `#/teens` へ遷移する
- `route-change` イベントには `path` に加えて `params` を含める

### 2.4 ローディング / エラー表示
- 動的 import の待機中はメイン領域にローディングインジケーターを表示する
- 読み込み失敗時はページ単位のエラーメッセージを表示し、ホームへ戻る導線を残す
- ゲームプレイ画面では従来どおりヘッダー / フッターを非表示にする

## 3. レイアウト設計

### 3.1 共通レイアウト構造

```text
Header
|- Logo + Penguin mascot
|- Site mode switch
|- Global navigation
`- Login / user status

Main
|- Page specific hero
|- Search + keyword area
|- Category rail
|- Feature cards / content list
`- Side panels or guide blocks

Footer
|- Copyright
|- Parents / Guide / Privacy / Contact links
`- Safety notes
```

### 3.2 モード別ビジュアル方針

| モード | 方向性 | UI要素 |
|--------|--------|--------|
| キッズ | `Yahoo!きっず` を参照しつつ、ペンギンを前面に出した明るく丸いUI | 大きい検索欄、アイコン付きカテゴリ、やわらかい角丸、黄色 / ミント / コーラルの差し色 |
| ティーンズβ | 中高生向け情報ポータル寄りの落ち着いたUI | ネイビー基調、情報量多めのカード、ニュース / 音楽 / 進路の導線、丸みは残しつつコントラストを上げる |

### 3.3 共通レイアウト要件
- ヘッダーは sticky とし、スクロール中もサイト切替と主要導線を保持する
- キッズトップはペンギンマスコットと検索を最上段に置き、カテゴリへすぐ到達できる構成とする
- ティーンズβトップはヒーロー直下に話題カードとカテゴリを並べ、情報ポータル感を優先する
- フッターには保護者向け案内、使い方、掲載基準、プライバシー、お問い合わせを配置する

### 3.4 レスポンシブブレークポイント

| 名称 | 幅 | レイアウト |
|------|-----|----------|
| モバイル | 0px〜767px | 1カラム、モバイルメニュー、カテゴリは横スクロールまたは2列グリッド |
| タブレット | 768px〜1023px | 2カラム、ヒーローと特集を上下分割 |
| デスクトップ | 1024px〜 | 2〜3カラム、サイドパネル常設、フルナビ表示 |

### 3.5 モバイルナビゲーション
- 開閉式のハンバーガーメニューを利用する
- メニュー先頭にサイト切替、続いてトップ、ランキング、保護者向け案内、使い方、About、お問い合わせ、ログインを並べる
- 管理画面導線はログイン済み時のみ表示する

### 4.3 ティーンズβトップ `#/teens`

- 共通ゲーム導線: 現時点ではゲーム本体は共通ページへ遷移する

### 4.4 カテゴリページ `#/kids/:category` `#/teens/:category`

#### 共通仕様
- トップではカテゴリレールを主導線とし、カテゴリページでカテゴリ内検索を行う
- モードとカテゴリに応じてヒーロー文言、キーワード、カード一覧を切り替える
- 検索対象は当該モードかつ当該カテゴリ内のコンテンツに限定する
- 人気キーワードクリックも検索実行と見なし、履歴保存対象に含める
- 0件時は空状態カードを出し、`キーワードを変える` `トップへ戻る` 導線を出す

#### カテゴリページ構成

```text
Category hero
|- Breadcrumb
|- Category title + icon
`- Search box scoped to current category

Content area
|- Featured cards
|- Content list / game list
`- Related keywords

Sidebar
|- Safety or guide links
`- Personalized recommendations
```

### 4.5 ゲーム説明ページ `#/games/typing`

| 領域 | 内容 |
|------|------|
| ヘッダー導線 | 直前のカテゴリまたはモードトップへ戻る、ランキング、ログイン |
| メイン | ゲーム概要、遊び方、モード説明、難易度説明、スクリーンショット枠 |
| CTA | `ゲームを始める` ボタンで `#/games/typing/play` へ遷移 |
| 関連情報 | 自己ベスト、最近のプレイ結果、関連カテゴリへの戻り導線 |

#### 戻り導線仕様
- ポータル配下からゲーム説明ページへ遷移する際に `lastPortalRoute` と `siteMode` を保持する
- ゲーム説明ページ、ゲーム結果画面、リザルト導線は `lastPortalRoute` がある場合そのルートへ戻す
- 直接アクセスやリロードで `lastPortalRoute` が失われた場合は、保持済み `siteMode` のトップへ戻す

### 4.6 ランキングページ `#/ranking`

#### 基本方針
- 当面は端末内の擬似ランキングを表示する
- `localStorage` に保存されたプレイ履歴を正本データとし、ランキングはその場で再計算する
- キッズ / ティーンズβのどちらから来たプレイかも保持し、表示切替に使えるようにする
- ランキング専用の永続キーは持たず、順位履歴が必要な場合はプレイ記録内のスナップショットとして保持する

#### プレイ履歴データ構造（ランキング算出用）

```json
{
  "id": "record-001",
  "gameId": "typing",
  "gameMode": "time-limit",
  "siteMode": "kids",
  "userName": "ぺんた",
  "score": 9800,
  "wpm": 85.2,
  "accuracy": 98.5,
  "rankSnapshot": 1,
  "createdAt": "2026-04-15T10:20:00+09:00",
  "deletedAt": null
}
```

### 4.7 ログイン / パーソナライズ `#/login`

#### モック認証仕様

| 項目 | 内容 |
|------|------|
| ユーザー名 | 任意文字列。空文字不可 |
| パスワード | 任意文字列。空文字不可 |
| 保存先 | `localStorage` |
| 保存対象 | 表示名、ログイン状態、モード嗜好、検索履歴の参照キー |
| 非保存対象 | 生パスワード |
| ログアウト | 表示状態を解除するのみ。履歴や成績は保持 |

#### 検索履歴データ構造

```json
{
  "id": "history-001",
  "mode": "kids",
  "route": "/kids/games",
  "category": "games",
  "query": "タイピング",
  "trigger": "submit",
  "createdAt": "2026-04-15T10:25:00+09:00",
  "deletedAt": null,
  "deletedBy": null
}
```

#### 特集最適化データ構造

```json
{
  "mode": "kids",
  "preferredCategories": ["games", "learn"],
```

```javascript
export function buildFeaturedOrder({ history, records, mode, defaults }) {
  const scoreMap = new Map();

  for (const item of defaults) {
    scoreMap.set(item.id, item.baseScore ?? 0);
  }

  // 検索履歴とプレイ履歴からカテゴリごとに重み付け
  // kids は安全導線を必ず残す
  // teens はニュース / 音楽 / 進路に加点

  return defaults
    .map(item => ({ ...item, score: scoreMap.get(item.id) ?? 0 }))
    .sort((a, b) => b.score - a.score);
}
```

## 6. データ設計 / 状態管理

### 6.1 コンテンツデータ方針
- ゲーム一覧データとトップ表示データを分離する
- モード別トップ、カテゴリ別一覧、キーワード群、特集カードは JSON または ES Module データとして保持する
- キッズとティーンズβは同一ゲーム一覧を参照しつつ、見せ方と導線を切り替える

### 6.2 ストレージ設計

| データ | 保存先 | キー | 用途 |
|--------|--------|------|------|
| サイトモード | `localStorage` | `penguin-site-mode` | 最終選択モードの復元 |
| モック認証 | `localStorage` | `penguin-games-auth` | ユーザー表示名、ログイン状態 |
| 検索履歴 | `localStorage` | `penguin-search-history` | 特集最適化、履歴表示、論理削除 |
| 特集最適化データ | `localStorage` | `penguin-featured-profile` | おすすめ順の保存 |
| プレイ履歴 / 成績 / ランキング履歴 | `localStorage` | `penguin-game-records` | 自己ベスト、プレイ履歴、ランキング算出、順位スナップショット |
| 直前ポータルルート | メモリ | なし | ゲーム説明 / 結果画面からカテゴリまたはトップへ戻す |
| 現在ルート | メモリ | なし | 画面切替用の一時状態 |

### 6.3 論理削除ルール

| 対象 | 削除方法 | 表示上の扱い | 備考 |
|------|----------|--------------|------|

| 特集最適化データ | リセットフラグ付けまたは再生成 | 次回表示時に再計算 | 実データは再構築可能 |
| プレイ履歴 / ランキング履歴 | `deletedAt` を設定 | 通常画面から除外 | 本格監査は DB 導入後 |

## 10. セキュリティ設計

### 10.1 フロントエンド対策

| 対策 | 内容 |
|------|------|
| XSS防止 | DOM 反映は `textContent` 優先、URL はホワイトリストまたはエスケープで扱う |
| CSP | スクリプト、スタイル、iframe、画像の読み込み元を制限する |
| モック認証 | 生パスワードは保存しない。表示名とログイン状態のみ保持する |
| 外部リンク注意 | 外部サイト利用前に注意ページへ誘導できる構造を持つ |
| データ削除 | 画面からは論理削除し、将来監査に備えた形式を維持する |
| お問い合わせ | Google Forms の iframe `sandbox` を維持する |

### 10.2 将来の外部検索フィルタ方針

| モード | 許容範囲 | 除外対象 |
|--------|----------|----------|
| キッズ | 学習、図鑑、ゲーム、子ども向けニュース | 性的表現、過度な暴力、危険行為、SNS誘導、マルウェア、詐欺 |
| ティーンズβ | ニュース、音楽、学び、トレンド、一般的なSNS導線 | 性的表現、過度な暴力、マルウェア、詐欺 |

### 10.3 プライバシー方針への反映事項
- `localStorage` に保存するデータ種類を明示する
- 検索履歴とおすすめ最適化の削除方法を明示する
- モック実装では端末内保持のみであることを明示する
- 本格的な管理者監査は DB 導入後の別要件であることを明示する

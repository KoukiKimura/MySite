## 1. ゲーム概要

### 1.1 コンセプト
ローマ字タイピングの練習・腕試しができるブラウザゲーム。日本語の単語や文章をローマ字で素早く正確にタイピングし、スコアを競う。

### 1.2 対象ユーザー
- タイピングを練習したい初心者〜中級者
- ローマ字入力を学ぶ小中学生
- タイピング速度を競いたいユーザー

### 1.3 基本仕様

| 項目 | 内容 |
|------|------|
| ジャンル | ローマ字タイピング |
| 描画方式 | HTML5 Canvas |
| 入力方式 | 物理キーボード（PC推奨） |
| プレイ人数 | 1人 |

#### モバイル対応方針
- タイピングゲームは**PC（物理キーボード）を推奨**とする
- モバイルアクセス時はゲーム説明ページで「PC推奨」のメッセージを表示
- モバイルでの「プレイ開始」ボタンは非活性とし、注意喚起テキストを表示する
- 将来的にモバイル向けフリック入力モードを追加する余地は残す（§8参照）

#### ScoreCalculator — スコア計算

```javascript
// domain/services/score-calculator.js
/**
 * スコア計算ルール
 *
 * 基本点: 正タイプ1文字 = 10点
 * コンボボーナス: combo >= 5 で 1.5倍、>= 10 で 2.0倍、>= 20 で 3.0倍
 * ミスペナルティ: ミス1回 = -5点（最低0点）
 * WPM算出: (正タイプ文字数 / 5) / (経過時間(分))
 * 正確率: 正タイプ数 / 総キーストローク数 * 100
 */
```

##### スコア計算式

> **コンボボーナスの適用単位**: キーストロークごとに適用。
> 正タイプ1打鍵ごとに `10 × その時点のコンボ倍率` を加算する。
> コンボ倍率は正解打鍵数（ミスでリセット）に基づく。

| 項目 | 計算式 |
|------|--------|
| 基本スコア | `correctCount × 10` |
| コンボボーナス | `基本スコア × コンボ倍率` |
| ミス減点 | `missCount × 5` |
| 最終スコア | `max(0, 基本スコア + コンボボーナス - ミス減点)` |
| WPM | `(correctCount / 5) / (elapsedTime / 60000)` |
| 正確率(%) | `correctCount / totalKeystrokes × 100` |

##### コンボ倍率テーブル

| コンボ数 | 倍率 | 表示 |
|---------|------|------|
| 0〜4 | ×1.0 | — |
| 5〜9 | ×1.5 | 🔥 |
| 10〜19 | ×2.0 | 🔥🔥 |
| 20〜 | ×3.0 | 🔥🔥🔥 |

#### QuestionProvider — 出題管理

> **配置層**: `application/services/`（インフラ層のデータ読み込みに依存するためドメイン層ではなくアプリケーション層に配置）

```javascript
// application/services/question-provider.js
/**
 * 出題データの読み込みとシャッフル
 *
 * 1. QuestionRepository（インフラ層）を通じてJSONファイルを読み込み
 * 2. Fisher-Yatesアルゴリズムでシャッフル
 * 3. 必要数だけ切り出して返却
 *
 * 依存: QuestionRepository（インターフェースをapplication層で定義、実装はinfrastructure層）
 */
```

### 2.5 アプリケーション層（ユースケース）

#### ユースケース一覧

| No | ユースケース | トリガー | 説明 |
|----|------------|---------|------|
| UC-1 | ゲーム開始 | メニューで「スタート」選択 | 設定に基づきセッション生成、カウントダウン開始 |
| UC-2 | キー入力処理 | キーボード押下 | 正誤判定、スコア更新、次の文字/問題へ進行 |
| UC-3 | ゲーム終了 | 時間切れ or 全問完了 | スコア確定、リザルト画面表示 |
| UC-4 | リトライ | リザルト画面で「リトライ」選択 | 同じ設定で新セッション生成 |
| UC-5 | メニューに戻る | リザルト画面で「メニュー」選択 | ゲーム内メニュー画面（G-1）に遷移。サイトルーターは使用せず、ゲームモジュール内の画面切替 |

#### UC-2: キー入力処理（詳細フロー）

```
キーボード押下
    │
    ├─ ゲームに関係ないキー → 無視
    │
    └─ 英字キー
        │
        ├─ 現在位置のローマ字パターンと照合
        │
        ├─ 一致 → 正解処理
        │   ├─ 入力位置を1つ進める
        │   ├─ correctCount++
        │   ├─ combo++
        │   ├─ 現在の問題の全文字完了？
        │   │   ├─ Yes → 次の問題へ（or ゲーム終了判定）
        │   │   └─ No  → 次の文字待ち
        │   └─ スコア再計算
        │
        └─ 不一致 → ミス処理
            ├─ missCount++
            ├─ combo = 0（リセット）
            ├─ ミスエフェクト表示
            └─ スコア再計算
```

## 4. Canvas描画設計

### 4.1 Canvas構成

> **描画方式の方針**:
> - **プレイ画面 (G-3)**: Canvas描画（リアルタイム更新が必要なため）
> - **メニュー画面 (G-1)、リザルト画面 (G-5)**: HTML/CSS + DOM で構築（ボタン操作やアクセシビリティに有利）
> - **カウントダウン (G-2)、一時停止 (G-4)**: Canvas上のオーバーレイ描画
>
> メニュー・リザルト画面はDOM要素として実装し、プレイ開始時にCanvas要素へ切り替える。

```javascript
// presentation/canvas/typing-canvas.js

/**
 * Canvas設定
 * - 論理サイズ: 800 x 600（アスペクト比 4:3固定）
 * - 物理サイズ: ウィンドウサイズに応じてスケーリング
 * - デバイスピクセル比対応（Retina等）
 */
const CANVAS_WIDTH = 800;
const CANVAS_HEIGHT = 600;
```

### 4.2 描画レイヤー

| レイヤー | 描画内容 | 更新頻度 |
|---------|---------|---------|
| 背景レイヤー | 背景色・背景装飾 | 画面切り替え時 |
| UIレイヤー | スコア、タイマー、進捗バー | 毎フレーム |
| テキストレイヤー | 出題テキスト、ローマ字ガイド | 問題切り替え時 |
| 入力レイヤー | 入力状態の色分け表示 | キー入力時 |
| エフェクトレイヤー | コンボ演出、ミスフラッシュ | イベント発生時 |

### 4.3 アニメーションループ

```javascript
// presentation/canvas/game-loop.js

/**
 * requestAnimationFrame ベースのゲームループ
 *
 * 更新サイクル:
 * 1. deltaTime計算
 * 2. ゲーム状態更新（タイマー等）
 * 3. エフェクト更新
 * 4. Canvas描画
 */
function gameLoop(timestamp) {
  const deltaTime = timestamp - lastTimestamp;
  lastTimestamp = timestamp;

  update(deltaTime);   // 状態更新
  render();            // 描画

  if (session.state === 'PLAYING') {
    requestAnimationFrame(gameLoop);
  }
}
```

### 4.4 エフェクト仕様

| エフェクト | トリガー | 演出 | 時間 |
|-----------|---------|------|------|
| 正解フラッシュ | 1問完了 | テキスト領域が緑にフラッシュ | 200ms |
| ミスフラッシュ | ミスタイプ | ミス文字が赤にフラッシュ + 画面振動 | 150ms |
| コンボ表示 | コンボ5以上 | 🔥アイコン + 倍率テキストアニメ | 常時表示 |
| カウントダウン | ゲーム開始前 | 数字の拡大→フェード | 各1秒 |
| ランクイン演出 | リザルト表示 | ランクが大きく表示 + パーティクル | 1.5秒 |

## 5. 入力処理設計

### 5.1 キーイベント処理

```javascript
// presentation/input/keyboard-handler.js

/**
 * キーボード入力処理
 *
 * 使用イベント: keydown
 * - keydown を使用（keypress は非推奨）
 * - event.key を使用（event.keyCode は非推奨）
 * - IMEオン時の入力は無視（event.isComposing チェック）
 */
```

#### 処理対象キー

| キー | 処理 |
|------|------|
| `a`〜`z` | タイピング入力（小文字に統一） |
| `A`〜`Z` | 小文字に変換して処理 |
| `-` | ローマ字の長音（ー）に対応 |
| `Escape` | 一時停止トグル |
| その他 | 無視 |

### 5.2 ローマ字照合アルゴリズム

```
入力: "densya" を打ちたい場合（「でんしゃ」）

文字分解:
  で → "de"
  ん → "n" (次が子音のため) or "nn"
  しゃ → "sya" or "sha"

許容パターン:
  "densya", "densha", "dennsya", "dennsha"

照合方式:
  1. 現在位置から各パターンの先頭一致を確認
  2. 複数パターンが候補にある場合は、打鍵が進むにつれ候補を絞る
  3. どのパターンにも一致しなければミス判定
```

#### 照合のデータ構造

```javascript
/**
 * 各「かな」に対して、許容されるローマ字入力パターンの配列を保持
 *
 * 例: "でんしゃ" の場合
 * [
 *   { kana: "で",  patterns: ["de"] },
 *   { kana: "ん",  patterns: ["n", "nn"] },   // 次が"し"(子音)なので"n"でOK
 *   { kana: "しゃ", patterns: ["sya", "sha"] },
 * ]
 *
 * 入力時:
 *   - 現在のかなチャンクの全patternsを候補として保持
 *   - 1文字入力するたびに先頭一致で候補を絞り込み
 *   - 候補が1つに確定 or 全文字入力完了 → 次のかなチャンクへ
 *   - 候補が0 → ミス判定
 */
```

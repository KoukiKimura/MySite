## 1. ゲーム概要

### 1.1 コンセプト
マウスでバーを操り、5つのステージを順に攻略するシンプルなブロック崩し。短時間で遊べるが、スコアとコンボで繰り返し挑戦したくなる設計とする。

### 1.2 基本仕様

| 項目 | 内容 |
|------|------|
| ジャンル | ブロック崩し |
| 描画方式 | HTML5 Canvas |
| 入力方式 | マウス / ポインター |
| プレイ人数 | 1人 |
| ステージ数 | 5 (`1-1`〜`1-5`) |
| 初期残機 | 3 |

### 1.3 ゲームループ
1. 説明ページからプレイページへ遷移する
2. 開始オーバーレイでゲームを開始する
3. バーを動かしてボールを反射し、ブロックを破壊する
4. ステージクリアで次ステージへ進む
5. 残機 0 でゲームオーバー、`1-5` クリアでゲームクリアとする

## 2. DDD ドメイン設計

### 2.1 コンテキスト構成

```
games/breakout/
│
├── domain/              ゲームルール
├── application/         ユースケース
├── infrastructure/      ステージ定義供給
└── presentation/        Canvas 描画と入力
```

### 2.2 エンティティ

#### GameSession
- ゲーム全体の進行状態を保持する
- 主な責務:
  - 現在ステージ
  - 残機
  - スコア
  - 現在コンボと最大コンボ
  - READY / PLAYING / PAUSED / FINISHED の遷移

#### StageState
- 1ステージ分の実行状態を保持する
- 主な責務:
  - ステージID
  - ブロック配置
  - 破壊済み状態
  - ボール速度、バー幅などのステージパラメータ

### 2.3 ドメインサービス

#### ScoreCalculator
- ブロック破壊時の得点を計算する
- 基本点 100 点に、現在コンボ数に応じたボーナスを加算する
- 想定式:
  - `score = 100 + combo * 25`

## 3. 画面設計

### 3.1 画面一覧

| 画面ID | 画面名 | 概要 |
|--------|--------|------|
| B-1 | 説明ページ | 遊び方、ルール、開始導線 |
| B-2 | プレイ画面 | Canvas 上でゲーム本編を表示 |
| B-3 | 一時停止オーバーレイ | 再開、やり直し、説明ページへ戻る |
| B-4 | 結果画面 | ゲームオーバー / クリア結果を表示 |

### 3.2 プレイ画面要素
- 上部 HUD:
  - ステージ番号
  - スコア
  - 残機
  - コンボ
- 中央プレイエリア:
  - ブロック群
  - ボール
  - バー
- 補助表示:
  - READY / STAGE CLEAR / GAME OVER / ALL CLEAR メッセージ

### 3.3 状態遷移

```
menu
  ↓ start
serve
  ↓ auto launch
playing
  ├─ ball lost & lives > 0 → serve
  ├─ stage clear & next exists → stage transition → serve
  ├─ lives = 0 → result(game over)
  └─ last stage clear → result(game clear)
```

## 4. ステージ設計

### 4.1 ステージ方針
- `1-1`: 基本操作確認
- `1-2`: 中央に穴のある配置
- `1-3`: 左右非対称の崩し順を試す配置
- `1-4`: 段差が多く反射角を使う配置
- `1-5`: 高密度の最終ステージ

### 4.2 難度調整
- ステージが進むほど以下を段階的に変更する
  - ボール速度上昇
  - バー幅縮小
  - ブロック配置密度上昇

---

## 5. データ設計

### 5.1 ステージ定義
- ステージデータは `infrastructure/data/stages.js` で管理する
- 1ステージは以下を持つ
  - `id`
  - `label`
  - `ballSpeed`
  - `paddleWidth`
  - `rows` ブロック配置
  - `theme` 描画色

### 5.2 ブロック配置表現
- `rows` は文字列配列で表す
- `1` はブロックあり、`0` は空きマス
- 実行時に StageState へ変換し、各ブロックへ一意 ID を付与する

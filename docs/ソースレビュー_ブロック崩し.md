## 2. 主な確認結果

| 観点 | 結果 |
|------|------|
| DDD レイヤー分離 | `domain` `application` `infrastructure` `presentation` の責務を分離済み |
| ステージ定義 | `infrastructure/data/stages.js` に `1-1`〜`1-5` を定義済み |
| 残機制 | `loseBall()` に集約し、残機 0 で `game-over` へ遷移 |
| コンボ加点 | `recordBlockDestroyed()` と `recordPaddleBounce()` へ集約 |
| 画面破棄 | `destroy()` で RAF、キーイベント、ポインターイベントを解放 |
| ルーティング | `#/games/breakout` と `#/games/breakout/play` を追加済み |
| ポータル一覧 | `games.json` に公開ステータスで追加済み |

---

## 3. 指摘事項と対応

| No | 指摘 | 対応 |
|----|------|------|
| 1 | ゲーム終了フレームで `requestAnimationFrame` が再登録される余地がある | `update()` 後に `destroyed` ガードを追加 |
| 2 | 本番ビルド時に新規ルートが解決されるか要確認 | `npm run build` で動的 import を検証 |
| 3 | コンボの二重加点リスク | StageState の `destroyed` フラグと `markBlockDestroyed()` の戻り値で重複破壊を防止 |
| 4 | 一時停止時の入力・描画継続リスク | `pause()` で RAF をキャンセルし、`resume()` で再開する構成を確認 |

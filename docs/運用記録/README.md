# 運用記録の置き場

## 目的

デプロイ記録、障害記録、作業記録などの運用ドキュメントを設計書と分けて管理する。

## 配置ルール

- 運用記録は `docs/運用記録/{対象}/` に置く。
- `{対象}` は `サイト` `タイピング` `ブロック崩し` など対象単位で分ける。
- リリースや障害対応に関する事実経緯を残し、仕様そのものは要件定義書・設計書へ反映する。

## 命名規則

| 種別 | 命名パターン | 例 |
|------|--------------|----|
| デプロイ記録 | `デプロイ記録_{対象}.md` | `デプロイ記録_ブロック崩し.md` |
| 障害記録 | `障害記録_{対象}.md` | `障害記録_サイト.md` |
| 作業記録 | `作業記録_{対象}.md` | `作業記録_タイピング.md` |

## パス例

- `docs/運用記録/ブロック崩し/デプロイ記録_ブロック崩し.md`
- `docs/運用記録/サイト/障害記録_サイト.md`
- `docs/運用記録/タイピング/作業記録_タイピング.md`

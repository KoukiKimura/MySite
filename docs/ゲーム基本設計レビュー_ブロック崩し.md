## 1. レビュー観点

- 要件との整合性
- 既存 SPA 構成との整合性
- DDD レイヤー分割の妥当性
- プレイ継続性と難易度バランス
- 実装・テスト可能性

---

## 2. 指摘事項と対応

| No | 指摘 | 対応方針 |
|----|------|----------|
| 1 | ボール消失後のステージ扱いが曖昧 | 破壊済みブロックは維持し、同一ステージ継続と定義 |
| 2 | コンボ加点の解除条件が曖昧 | バー接触、ボール消失、ステージ遷移でリセットと明記 |
| 3 | ステージ差分が配置だけでは弱い | ボール速度とバー幅もステージ要素へ追加 |
| 4 | モバイル時の操作保証が弱い | PC 推奨としつつ Pointer Events で最低限の追従は可能にする |
| 5 | プレイ中離脱時の導線が不足 | 一時停止オーバーレイに再開・再挑戦・戻るを追加 |

---

## 3. レビュー結論

- 要件で求められた5ステージ制、残機制、コンボ得点制は基本設計で表現できている
- 既存の `games/typing/` と同等の責務分割で `games/breakout/` を構成できる
- プレイロジックの中心を GameSession と StageState に寄せることで、描画コードとの分離が可能
- 詳細設計では衝突判定の粒度、`destroy()` の責務、テスト対象関数を具体化する

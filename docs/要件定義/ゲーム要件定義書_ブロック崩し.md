## 1. ゲーム機能要件

### 1.1 基本仕様
| 項目 | 内容 |
|------|------|
| ジャンル | ブロック崩し（アーケード） |
| 描画方式 | HTML5 Canvas |
| 入力方式 | マウス操作（PC推奨）、タッチ操作（モバイル対応）、キーボード（←→キー） |
| プレイ人数 | 1人 |

### 1.2 難易度設定
| 難易度 | ボール速度 | パドル幅 | ブロック数 | 特記 |
|--------|----------|---------|----------|------|
| 初級 | 低速 | 広め | 少なめ | パワーアップ頻度高 |
| 中級 | 中速 | 標準 | 標準 | 標準構成 |
| 上級 | 高速 | 狭め | 多め | パワーアップ頻度低 |

### 1.3 ステージ構成
- JSONファイルでステージデータを管理
- 難易度ごとに複数ステージを用意
- ブロック配置パターンはステージデータで定義
- ステージクリアで次ステージへ進行（全ステージクリアでゲーム完了）

### 1.4 ブロック種別
| 種別 | 概要 |
|------|------|
| 通常ブロック | 1ヒットで破壊 |
| 硬いブロック | 複数ヒット必要（ヒット数はステージデータで定義） |
| 無敵ブロック | 破壊不可（ステージ構造の一部） |
| パワーアップブロック | 破壊時にパワーアップアイテムをドロップ |

### 1.5 パワーアップアイテム
| アイテム | 効果 |
|---------|------|
| パドル拡大 | パドル幅が一時的に拡大 |
| マルチボール | ボールが2個に増加 |
| スロー | ボール速度が一時的に低下 |
| レーザー | パドルからレーザーを発射してブロックを直接破壊 |
| ライフアップ | 残機が1増加（最大5） |

### 1.6 ゲームフロー
```
1. ゲームメニュー画面（難易度・ステージ選択）
       ↓
2. カウントダウン（3, 2, 1, Start!）
       ↓
3. ゲームプレイ（パドル操作 → ボール反射 → ブロック破壊 → パワーアップ獲得）
       ↓
4a. ステージクリア → 次ステージへ（全クリアでゲーム終了へ）
4b. ボールアウト → 残機減少 → 残機0でゲームオーバー
       ↓
5. リザルト画面（スコア、クリアステージ数、獲得アイテム等）
       ↓
6. リトライ / メニューに戻る
```

### 1.7 画面表示要素
- ブロックエリア（上部、ステージデータに基づいて配置）
- ボール（物理的な跳ね返り）
- パドル（下部、プレイヤー操作）
- パワーアップアイテム（落下中）
- 残機表示（ペンギンアイコン × 残機数）
- 現在スコア
- 残りステージ進捗

### 1.8 スコアリング
| 要素 | ポイント |
|------|---------|
| 通常ブロック破壊 | 10pt |
| 硬いブロック破壊 | 30pt |
| パワーアップ獲得 | 50pt |
| ステージクリアボーナス | 残機数 × 200pt |
| コンボボーナス | 連続破壊数 × 5pt 加算 |

## 3. DDD ゲームコンテキスト

### 3.1 値オブジェクト (Value Objects)

| 名前 | ファイル | 概要 |
|------|---------|------|
| `Vector2` | `domain/values/vector2.js` | 2次元ベクトル（位置・速度） |
| `Difficulty` | `domain/values/difficulty.js` | 難易度定数（EASY / NORMAL / HARD） |
| `BlockType` | `domain/values/block-type.js` | ブロック種別定数 |
| `PowerUpType` | `domain/values/power-up-type.js` | パワーアップ種別定数 |
| `Score` | `domain/values/score.js` | スコア集計（不変オブジェクト） |
| `BallState` | `domain/values/ball-state.js` | ボール状態（ACTIVE / LOST / HELD） |

### 3.2 エンティティ (Entities)

| 名前 | ファイル | 概要 |
|------|---------|------|
| `Ball` | `domain/entities/ball.js` | ボールの位置・速度・状態 |
| `Paddle` | `domain/entities/paddle.js` | パドルの位置・サイズ |
| `Block` | `domain/entities/block.js` | ブロックの位置・種別・耐久値 |
| `PowerUpItem` | `domain/entities/power-up-item.js` | 落下中のパワーアップアイテム |
| `Stage` | `domain/entities/stage.js` | ステージデータ（ブロック配置） |
| `GameSession` | `domain/entities/game-session.js` | ゲームセッション（スコア・残機・進行） |

### 3.3 ドメインサービス (Domain Services)

| 名前 | ファイル | 概要 |
|------|---------|------|
| `CollisionDetector` | `domain/services/collision-detector.js` | ボール・パドル・ブロック間の衝突判定 |
| `ScoreCalculator` | `domain/services/score-calculator.js` | スコア計算ロジック |
| `StageBuilder` | `domain/services/stage-builder.js` | JSONからStageエンティティを構築 |
| `PhysicsEngine` | `domain/services/physics-engine.js` | ボールの物理演算（速度・反射） |

### 3.4 ユースケース (Use Cases)

| ID | 名前 | ファイル | 概要 |
|----|------|---------|------|
| UC-1 | StartGame | `application/use-cases/start-game.js` | ゲームセッション初期化・ステージ読み込み |
| UC-2 | UpdateFrame | `application/use-cases/update-frame.js` | 1フレーム分の物理演算・衝突処理 |
| UC-3 | MovePaddle | `application/use-cases/move-paddle.js` | パドル位置更新 |
| UC-4 | ApplyPowerUp | `application/use-cases/apply-power-up.js` | パワーアップアイテム効果適用 |
| UC-5 | LoseLife | `application/use-cases/lose-life.js` | 残機減少・ゲームオーバー判定 |
| UC-6 | ClearStage | `application/use-cases/clear-stage.js` | ステージクリア判定・次ステージ遷移 |
| UC-7 | FinishGame | `application/use-cases/finish-game.js` | ゲーム完了・スコア確定 |

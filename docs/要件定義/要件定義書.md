## 1. プロジェクト概要

### 1.1 目的
ConoHa Wingレンタルサーバー上に、複数のHTML5ゲームを提供するポータルサイト「ペンギンげーむず！」を構築する。

### 1.2 初回スコープ
- キッズ向けポータル画面の作成（`Yahoo!きっず` 参照 + オリジナル要素追加）
- ティーンズβトップおよびカテゴリ画面モックの作成
- ローマ字タイピングゲーム 1本の作成
- ログイン、検索履歴、特集最適化、成績保持、ランキング保持のモック作成
- 保護者向け案内・安全関連ページ・管理画面の追加

### 1.3 副次目的
- 小中学生向けプログラミング教育資料（Moodle）のサンプルプロジェクトとして活用
- モダンかつ初心者が理解しやすいコード構成を維持する

### 1.4 設計方針
- **DDD（ドメイン駆動設計）戦略設計** を採用し、ビジネスドメインに基づいた境界づけられたコンテキストでモジュールを分割する
- モダンだが初心者が理解しやすいシンプルな構成
- 外部ライブラリ依存を最小限に
- ペンギンをマスコットキャラクターとし、キッズ向けは明るく視認性の高いUI、ティーンズβは情報ポータル寄りのUIとする
- 画像アセットは実装段階で別途生成プロンプトから作成する前提とする

## 4. 機能要件

### 4.1 ポータル画面（提供画面）

#### 4.1.1 キッズ向けトップページ
- `Yahoo!きっず` を参照しつつ、ペンギンをマスコットにしたオリジナルUIを構築する
- キッズ側の基本カテゴリは `特集 / 学習 / ゲーム / 図鑑 / タグ` を基準とし、オリジナルカテゴリや特集枠を追加できる構造とする
- カテゴリ名にはアイコンを併用し、視認性を向上させる
- 検索欄、人気キーワード、注目カード、ゲーム一覧、保護者向け導線をトップに配置する
- ゲーム追加が容易なデータ駆動型設計とし、ゲーム一覧データとトップ用コンテンツ定義を分離する

#### 4.1.2 ティーンズβトップページ — モック
- 中高生向け情報ポータル寄りのUIとし、ニュース、学び、音楽、放課後、進路などを扱う
- キッズ版より落ち着いた情報密度とし、同一ブランド内の別モードとして切り替えられる
- 現段階ではトップとカテゴリのモックを提供し、将来的に個別ページや検索連携を本実装する

#### 4.1.3 カテゴリページ
- モード別ルートを採用し、`#/kids/...` と `#/teens/...` でページを分ける
- 例: `#/kids/games`, `#/kids/learn`, `#/teens/news`, `#/teens/music`
- 各カテゴリページでは、そのカテゴリ内コンテンツのみを対象に検索する
- カテゴリページでも注目カード、一覧、関連キーワードを表示できる

#### 4.1.4 検索機能
- 現段階ではサイト内検索のモックとし、カテゴリ内コンテンツの絞り込みを行う
- 将来的には外部検索連携を実装し、モード別にフィルタ方針を切り替える
- キッズ検索では性的表現、過度な暴力、危険行為、SNS誘導、マルウェア/詐欺/不正サイトを除外対象とする
- ティーンズβ検索ではニュース、音楽、トレンド、SNSなど許容範囲を広げるが、性的表現、過度な暴力、マルウェア/詐欺は引き続き除外対象とする

#### 4.1.5 ログイン・パーソナライズ — モック
- ログイン / ログアウトUIとモック認証を作成する
- ログイン状態、サイトモード、検索履歴、おすすめ最適化データは当面 `localStorage` に保存する
- 検索履歴をもとにトップ画面の特集表示を最適化する
- 検索履歴やおすすめ関連データはユーザーが削除できるようにし、削除方式は論理削除とする
- 本格的なユーザー管理、端末間同期、権限制御はDB導入後の別要件とする

#### 4.1.6 成績・ランキング保持 — モック
- ゲーム成績、プレイ履歴、ランキング履歴を保存できる
- 当面は端末内の擬似ランキングとし、`localStorage` ベースで保持する
- 将来的にはDB/API連携により全体ランキングへ拡張する

#### 4.1.7 保護者・安全関連ページ
- 保護者向け案内ページを用意する
- 使い方ページを用意する
- 外部リンク注意ページを用意する
- コンテンツ掲載基準ページを用意する
- プライバシーポリシーおよび問い合わせ導線を用意する

#### 4.1.8 ゲーム説明ページ
- ゲームの詳細説明、遊び方、スクリーンショット
- 「ゲームを始める」ボタン → ゲーム画面へ遷移
- ゲームごとに個別ページ

#### 4.1.9 サイト紹介・About
- サイトの概要・目的の説明ページ
- 制作者情報（任意）

#### 4.1.10 お問い合わせ
- **方式: Google Forms埋め込み**
  - セキュリティ: サーバーサイド処理不要、スパム対策はGoogle側で対応
  - コスト: 無料
  - 実装: iframeでGoogle Formsを埋め込み、またはリンクで遷移
  - 代替案: Formspree（月50件無料）、メールリンク（mailto）

#### 4.1.11 共通UI要素
- ヘッダー（サイトロゴ、ナビゲーション、モード切替）
- サイト切替UI（`キッズ` / `ティーンズβ`）
- カテゴリアイコン
- フッター（コピーライト、リンク）
- レスポンシブナビゲーション（ハンバーガーメニュー等）

#### 4.1.12 管理画面 — モック
- `#/admin` に管理画面を追加する
- 現段階では管理機能のモックとし、運営向け導線や確認UIを先行定義する
- 論理削除済みの履歴・成績・ランキング履歴の管理者確認は、DB導入後の別要件で本実装する

### 4.2 タイピングゲーム

#### 4.2.1 基本仕様
| 項目 | 内容 |
|------|------|
| ジャンル | ローマ字タイピング |
| 描画方式 | HTML5 Canvas |
| 入力方式 | 物理キーボード（モバイルはソフトキーボード検討） |

#### 4.2.2 出題モード選択
| モード | 内容 |
|--------|------|
| 日本語→ローマ字モード | 日本語の単語・文章が表示され、ローマ字で入力 |
| 英語単語モード | 英単語が表示され、そのままタイピング |

#### 4.2.3 難易度設定
| 難易度 | 出題内容 | 制限時間 |
|--------|----------|----------|
| 初級 | 短い単語（2〜4文字程度） | 長め |
| 中級 | 標準的な単語〜短文 | 標準 |
| 上級 | 長文・難しい語彙 | 短め |

#### 4.2.4 ゲームモード
| モード | 内容 |
|--------|------|
| 時間制限モード | 制限時間（60秒等）内にできるだけ多くタイピング |
| 問題数固定モード | 固定問題数（10問・20問等）をこなすタイムを競う |

#### 4.2.5 ゲームフロー
```
1. ゲームメニュー画面（モード・難易度選択）
       ↓
2. カウントダウン（3, 2, 1, Start!）
       ↓
3. ゲームプレイ（出題 → 入力 → 正誤判定 → 次の問題）
       ↓
4. リザルト画面（スコア、正確率、WPM等）
       ↓
5. リトライ / メニューに戻る
```

#### 4.2.6 画面表示要素
- 出題テキスト（日本語表示 + ローマ字ガイド）
- 入力中のローマ字表示（正誤をリアルタイムで色分け）
- 残り時間 / 進捗バー
- 現在のスコア / コンボ数
- ミスタイプ表示

#### 4.2.7 スコアリング
- 正確にタイプした文字数ベース
- ミスタイプによる減点
- コンボボーナス（連続正解）
- WPM（Words Per Minute）算出

#### 4.2.8 出題データ
- JSON形式で出題データを管理
- 難易度別にデータファイルを分離
- 将来的にデータ追加が容易な構造

## 6. DDD戦略設計

### 6.1 ドメインモデル概要

本プロジェクトでは以下の境界づけられたコンテキスト（Bounded Context）を定義する。

```
┌──────────────────────────────────────────────────────────────────────┐
│                         ペンギンげーむず！                           │
│                                                                      │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐               │
│  │ ポータル     │  │ ゲーム       │  │ ユーザー     │               │
│  │ コンテキスト │  │ コンテキスト │  │ コンテキスト │               │
│  │ ・モード別TOP│  │ ・タイピング  │  │ ・認証       │               │
│  │ ・カテゴリ   │  │ ・出題/スコア │  │ ・履歴/おすすめ│             │
│  │ ・検索/案内  │  │              │  │ ・成績/ランキング│           │
│  └──────────────┘  └──────────────┘  └──────────────┘               │
│                                                                      │
│  ┌──────────────┐  ┌──────────────┐                                  │
│  │ 管理         │  │ 共有カーネル │                                  │
│  │ コンテキスト │  │              │                                  │
│  │ ・管理画面   │  │ ・共通UI     │                                  │
│  │ ・掲載基準   │  │ ・ルーティング│                                  │
│  │ ・監査ログ   │  │ ・状態管理   │                                  │
│  └──────────────┘  └──────────────┘                                  │
└──────────────────────────────────────────────────────────────────────┘
```

### 6.2 境界づけられたコンテキスト

| コンテキスト | 責務 | 今回のスコープ |
|-------------|------|---------------|
| ポータルコンテキスト | モード別トップ/カテゴリ、検索導線、安全案内、ゲーム一覧、サイト情報、お問い合わせ | 実装 + 改修 |
| ゲームコンテキスト | 個別ゲームのロジック・描画・入力処理 | タイピングゲームのみ実装 |
| ユーザーコンテキスト | 認証、検索履歴、特集最適化、成績、ランキング | モック |
| 管理コンテキスト | 管理画面、掲載基準、削除データ監査 | モック / 将来本実装 |
| 共有カーネル | Canvas基盤、共通UIコンポーネント、ルーティング、状態管理 | 実装 |

### 6.3 ゲームコンテキスト — タイピングゲームのドメインモデル

#### 値オブジェクト (Value Objects)
| 名称 | 説明 |
|------|------|
| `GameMode` | ゲームモード（時間制限 / 問題数固定） |
| `Difficulty` | 難易度（初級 / 中級 / 上級） |
| `QuestionMode` | 出題モード（日本語→ローマ字 / 英語単語） |
| `Score` | スコア値（正タイプ数、ミス数、コンボ等を含む） |
| `RomajiInput` | ローマ字入力の正誤判定結果 |

#### エンティティ (Entities)
| 名称 | 説明 |
|------|------|
| `GameSession` | 1回のゲームプレイセッション |
| `Question` | 1問の出題データ（表示テキスト + ローマ字解答） |

#### ドメインサービス (Domain Services)
| 名称 | 配置層 | 説明 |
|------|--------|------|
| `RomajiConverter` | domain/services | 日本語→ローマ字変換（複数入力パターン対応） |
| `ScoreCalculator` | domain/services | スコア計算ロジック（WPM、コンボ等） |
| `QuestionProvider` | application/services | 出題データの取得・シャッフル（インフラ層に依存するためアプリケーション層に配置） |

## 8. 非機能要件

### 8.1 パフォーマンス
| 指標 | 目標値 |
|------|--------|
| 初回ロード時間 | 3秒以内（LTE環境） |
| ゲーム内フレームレート | 60fps維持 |
| アセット最適化 | 画像圧縮、コードminify、Tree Shaking |

### 8.2 セキュリティ
- XSS対策（ユーザー入力のサニタイズ・エスケープ）
- CSP（Content Security Policy）ヘッダー設定
- 外部リソースの整合性チェック（SRI: Subresource Integrity）
- お問い合わせはGoogle Forms外部委託でサーバーサイドリスクを回避
- 将来的な外部検索連携では、モード別に安全フィルタポリシーを適用する
- キッズでは性的表現、過度な暴力、危険行為、SNS誘導、マルウェア/詐欺を強く遮断する
- ティーンズβでは一般的なニュース、音楽、トレンド、SNSを許容しつつ、性的表現、過度な暴力、マルウェア/詐欺は遮断する

### 8.3 アクセシビリティ
- セマンティックHTML
- キーボードナビゲーション対応
- 適切なコントラスト比（WCAG 2.1 AA準拠目標）

### 8.4 SEO
- メタタグ設定（title, description）
- OGP（Open Graph Protocol）設定
- 構造化データ（JSON-LD）

### 8.5 保守性
- DDD戦略設計に基づくモジュール分割
- コンテキスト間の疎結合
- 新規ゲーム追加が容易なプラグイン的構造
- 教育利用を考慮したコード可読性の維持

### 8.6 データ管理・プライバシー
- 現段階のログイン状態、サイトモード、検索履歴、特集最適化データ、成績、ランキング履歴は `localStorage` に保持する
- 検索履歴・おすすめ最適化データ・成績履歴の削除は論理削除とする
- 削除済みデータの管理者確認や監査は、DB導入後の別要件で本実装する

## 2. ドメイン詳細

### 2.1 GameSession

主要プロパティ:
- `state`
- `stages`
- `stageIndex`
- `currentStage`
- `score`
- `lives`
- `maxLives`
- `combo`
- `maxCombo`
- `result`

主要関数:
- `createGameSession(stages, options)`
- `startSession(session)`
- `pauseSession(session)`
- `resumeSession(session)`
- `recordBlockDestroyed(session, blockId)`
- `recordPaddleBounce(session)`
- `loseBall(session)`
- `advanceStage(session)`
- `finishSession(session, resultType)`

### 2.2 StageState

主要プロパティ:
- `id`
- `label`
- `ballSpeed`
- `paddleWidth`
- `theme`
- `blocks`

主要関数:
- `createStageState(stageDefinition)`
- `countRemainingBlocks(stage)`
- `hasRemainingBlocks(stage)`
- `markBlockDestroyed(stage, blockId)`

### 2.3 ScoreCalculator

```js
baseScore = 100;
comboBonus = currentCombo * 25;
total = baseScore + comboBonus;
```

- `currentCombo` は「今回の破壊前の連続数」とする
- 例:
  - 1個目: 100
  - 2個目: 125
  - 3個目: 150

## 4. プレゼンテーション詳細

### 4.1 Canvas
- 論理サイズは固定し、表示サイズは親要素に合わせてスケーリングする
- `requestAnimationFrame` でループする
- 1フレームの `dt` は上限を設け、長時間停止復帰時の暴走を防ぐ

### 4.2 PointerController
- `pointermove` を購読する
- キャンバス内相対座標を論理座標へ変換する
- `destroy()` でイベントを解除する

### 4.3 PlayScreen 内部状態
- `phase`: `serve` / `playing` / `stage-transition` / `finished`
- `ball`: `x`, `y`, `vx`, `vy`, `radius`
- `paddle`: `x`, `y`, `width`, `height`, `targetX`

### 4.4 ボール更新アルゴリズム
1. `dt` からサブステップ数を算出する
2. 各サブステップでボールを移動する
3. 壁衝突を判定する
4. バー衝突を判定し、反射角を調整する
5. ブロック衝突を1件ずつ処理する
6. 下端到達時は `loseBall(session)` を呼ぶ
7. ブロック残数 0 で `advanceStage(session)` または `finishSession()` を呼ぶ

### 4.5 反射仕様
- 壁:
  - 左右で `vx` 反転
  - 上端で `vy` 反転
- バー:
  - 接触位置に応じて角度を変える
  - 反射時に `recordPaddleBounce(session)` を呼ぶ
- ブロック:
  - 侵入方向に応じて `vx` または `vy` を反転
  - `recordBlockDestroyed(session, blockId)` を呼ぶ

## 1. 目的

### 1.1 追加目的
- ポータル内の第2ゲームとして、短時間で遊べるアクションゲームを追加する
- キーボード中心のタイピングゲームに対し、マウス主体のゲーム体験を提供する
- ステージ攻略とスコアアタックの両方を楽しめる構成にする

### 1.2 初回スコープ
- シンプルなブロック崩しを 1-1 から 1-5 までの5ステージ構成で実装する
- バーはマウス追従で左右移動する
- ボールは壁、バー、ブロックで反射する
- ブロックを全破壊すると次ステージへ進む
- 残機が 0 になるとゲームオーバーとなり、次回再開は 1-1 からとする

### 1.3 スコープ外
- アイテムドロップ
- 複数ボール
- ブロック耐久値の多段階化
- ランキング保存
- BGM/SE の本格実装

## 2. 機能要件

### 2.1 ゲーム開始
- ゲーム説明ページから `#/games/breakout/play` へ遷移できること
- プレイページには開始オーバーレイを表示し、ユーザーが開始操作できること
- ゲーム開始時はステージ `1-1`、残機 `3`、スコア `0` で初期化すること

### 2.2 基本操作
- バーはマウスまたはポインター位置に追従して左右移動すること
- バーの中心がポインター位置に合うように移動すること
- バー位置はプレイエリア外へ出ないこと

### 2.3 ボールとライフ
- 常時アクティブなボールは1個とする
- ボールがバーより下へ落下した場合、そのボールは消失扱いとする
- ボール消失時に残機を1減算すること
- 残機が残っている場合は、同一ステージ・同一スコア・破壊済みブロック維持で再開すること
- 残機が 0 になった場合はゲームオーバー画面へ遷移すること
- ゲームオーバー後に再プレイした場合は `1-1` から再開すること

### 2.4 ステージ進行
- ステージは `1-1`、`1-2`、`1-3`、`1-4`、`1-5` の順で進行すること
- 各ステージは異なるブロック配置を持つこと
- 各ステージの全ブロック破壊時に次ステージへ遷移すること
- `1-5` クリア時はゲームクリアとすること

### 2.5 スコアリング
- ブロックを1個破壊するごとに得点を加算すること
- 同じボールがバーに触れずに連続でブロックを破壊した場合、連続ボーナスを累積すること
- 連続ボーナスは「直前までの連続数」に応じて増加すること
- コンボはバー接触時、ボール消失時、ステージ遷移時にリセットすること
- 最終結果画面に総スコアと最大コンボを表示すること

### 2.6 画面要件
- 説明ページ、プレイ画面、結果画面を用意すること
- プレイ画面ではステージ、スコア、残機、現在コンボを常時表示すること
- 一時停止操作を受け付け、再開またはリスタートまたは説明ページ戻りを選べること

## 3. 非機能要件

### 3.1 パフォーマンス
- HTML5 Canvas で描画すること
- 60fps 相当の更新を目標とすること
- 1フレーム内での衝突判定漏れを減らすため、移動量に応じた分割更新を行うこと

### 3.2 対応環境
- PC ブラウザを主対象とする
- 最新の Chrome、Edge、Safari を主要対象とする
- モバイル端末でも表示崩れは避けるが、PC 推奨の案内を表示する

### 3.3 保守性
- `games/breakout/` 配下は `domain` `application` `infrastructure` `presentation` に分割する
- ステージ定義は静的データとして分離し、描画処理と混在させない
- スコア計算とライフ制御はドメインロジックとしてテスト可能にする

---

## 4. 受け入れ条件

- `#/games/breakout` から説明ページが開ける
- `#/games/breakout/play` でゲームを開始できる
- マウス移動でバーが追従する
- ボールが壁・バー・ブロックで反射する
- 5ステージを順に進行できる
- ボール消失で残機が減る
- 残機 0 でゲームオーバーになる
- ブロック破壊でスコア加算される
- バー未接触連続破壊でコンボボーナスが増える
- 最終結果画面で再挑戦すると `1-1` から始まる

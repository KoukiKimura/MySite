## 1. プロジェクト概要

### 1.1 目的
ConoHa Wingレンタルサーバー上に、複数のHTML5ゲームを提供するポータルサイト「ペンギンげーむず！」を構築する。

### 1.2 初回スコープ
- ゲームポータル画面（提供画面）の作成
- ローマ字タイピングゲーム 1本の作成

### 1.3 副次目的
- 小中学生向けプログラミング教育資料（Moodle）のサンプルプロジェクトとして活用
- モダンかつ初心者が理解しやすいコード構成を維持する

### 1.4 設計方針
- **DDD（ドメイン駆動設計）戦略設計** を採用し、ビジネスドメインに基づいた境界づけられたコンテキストでモジュールを分割する
- モダンだが初心者が理解しやすいシンプルな構成
- 外部ライブラリ依存を最小限に

## 4. 機能要件

### 4.1 ポータル画面（提供画面）

#### 4.1.1 ゲーム一覧表示（トップページ）
- ゲームカード形式で一覧表示（サムネイル + タイトル + 簡易説明）
- カードクリックでゲーム説明ページへ遷移
- ゲーム追加が容易なデータ駆動型設計（JSON設定ファイル）
- ポップ・ゲーム風デザイン（ペンギンキャラクター等）

#### 4.1.2 ゲーム説明ページ
- ゲームの詳細説明、遊び方、スクリーンショット
- 「ゲームを始める」ボタン → ゲーム画面へ遷移
- ゲームごとに個別ページ

#### 4.1.3 ユーザー認証（ログイン）— モック
- ログイン / ログアウトのUI画面のみ作成
- 実際の認証処理はなし（将来バックエンド実装時に接続）
- モックユーザーでログイン状態をシミュレート

#### 4.1.4 ランキング機能 — モック
- ランキング表示画面のUI作成
- ダミーデータでランキング表示
- 将来バックエンド接続時にAPIから取得する構造で設計

#### 4.1.5 サイト紹介・About
- サイトの概要・目的の説明ページ
- 制作者情報（任意）

#### 4.1.6 お問い合わせ
- **方式: Google Forms埋め込み**
  - セキュリティ: サーバーサイド処理不要、スパム対策はGoogle側で対応
  - コスト: 無料
  - 実装: iframeでGoogle Formsを埋め込み、またはリンクで遷移
  - 代替案: Formspree（月50件無料）、メールリンク（mailto）

#### 4.1.7 共通UI要素
- ヘッダー（サイトロゴ、ナビゲーション）
- フッター（コピーライト、リンク）
- レスポンシブナビゲーション（ハンバーガーメニュー等）

### 4.2 タイピングゲーム

#### 4.2.1 基本仕様
| 項目 | 内容 |
|------|------|
| ジャンル | ローマ字タイピング |
| 描画方式 | HTML5 Canvas |
| 入力方式 | 物理キーボード（モバイルはソフトキーボード検討） |

#### 4.2.2 出題モード選択
| モード | 内容 |
|--------|------|
| 日本語→ローマ字モード | 日本語の単語・文章が表示され、ローマ字で入力 |
| 英語単語モード | 英単語が表示され、そのままタイピング |

#### 4.2.3 難易度設定
| 難易度 | 出題内容 | 制限時間 |
|--------|----------|----------|
| 初級 | 短い単語（2〜4文字程度） | 長め |
| 中級 | 標準的な単語〜短文 | 標準 |
| 上級 | 長文・難しい語彙 | 短め |

#### 4.2.4 ゲームモード
| モード | 内容 |
|--------|------|
| 時間制限モード | 制限時間（60秒等）内にできるだけ多くタイピング |
| 問題数固定モード | 固定問題数（10問・20問等）をこなすタイムを競う |

#### 4.2.5 ゲームフロー
```
1. ゲームメニュー画面（モード・難易度選択）
       ↓
2. カウントダウン（3, 2, 1, Start!）
       ↓
3. ゲームプレイ（出題 → 入力 → 正誤判定 → 次の問題）
       ↓
4. リザルト画面（スコア、正確率、WPM等）
       ↓
5. リトライ / メニューに戻る
```

#### 4.2.6 画面表示要素
- 出題テキスト（日本語表示 + ローマ字ガイド）
- 入力中のローマ字表示（正誤をリアルタイムで色分け）
- 残り時間 / 進捗バー
- 現在のスコア / コンボ数
- ミスタイプ表示

#### 4.2.7 スコアリング
- 正確にタイプした文字数ベース
- ミスタイプによる減点
- コンボボーナス（連続正解）
- WPM（Words Per Minute）算出

#### 4.2.8 出題データ
- JSON形式で出題データを管理
- 難易度別にデータファイルを分離
- 将来的にデータ追加が容易な構造

## 6. DDD戦略設計

### 6.1 ドメインモデル概要

本プロジェクトでは以下の境界づけられたコンテキスト（Bounded Context）を定義する。

```
┌─────────────────────────────────────────────────────┐
│                   ペンギンげーむず！                    │
│                                                     │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────┐  │
│  │  ポータル     │  │  ゲーム      │  │  ユーザー  │  │
│  │  コンテキスト │  │  コンテキスト │  │ コンテキスト│  │
│  │              │  │              │  │           │  │
│  │ ・ゲーム一覧  │  │ ・タイピング  │  │ ・認証     │  │
│  │ ・サイト情報  │  │   ゲーム     │  │ ・プロフィル│  │
│  │ ・お問い合わせ│  │ ・出題管理   │  │ ・ランキング│  │
│  │              │  │ ・スコア計算  │  │           │  │
│  └──────────────┘  └──────────────┘  └───────────┘  │
│                                                     │
│  ┌──────────────┐                                   │
│  │  共有カーネル │                                   │
│  │              │                                   │
│  │ ・Canvas基盤 │                                   │
│  │ ・UIコンポー │                                   │
│  │   ネント     │                                   │
│  │ ・ルーティング│                                   │
│  └──────────────┘                                   │
└─────────────────────────────────────────────────────┘
```

### 6.2 境界づけられたコンテキスト

| コンテキスト | 責務 | 今回のスコープ |
|-------------|------|---------------|
| ポータルコンテキスト | ゲーム一覧管理、サイト情報、お問い合わせ | 実装 |
| ゲームコンテキスト | 個別ゲームのロジック・描画・入力処理 | タイピングゲームのみ実装 |
| ユーザーコンテキスト | 認証、プロフィール、ランキング | モック（UI のみ） |
| 共有カーネル | Canvas基盤、共通UIコンポーネント、ルーティング | 実装 |

### 6.3 ゲームコンテキスト — タイピングゲームのドメインモデル

#### 値オブジェクト (Value Objects)
| 名称 | 説明 |
|------|------|
| `GameMode` | ゲームモード（時間制限 / 問題数固定） |
| `Difficulty` | 難易度（初級 / 中級 / 上級） |
| `QuestionMode` | 出題モード（日本語→ローマ字 / 英語単語） |
| `Score` | スコア値（正タイプ数、ミス数、コンボ等を含む） |
| `RomajiInput` | ローマ字入力の正誤判定結果 |

#### エンティティ (Entities)
| 名称 | 説明 |
|------|------|
| `GameSession` | 1回のゲームプレイセッション |
| `Question` | 1問の出題データ（表示テキスト + ローマ字解答） |

#### ドメインサービス (Domain Services)
| 名称 | 説明 |
|------|------|
| `RomajiConverter` | 日本語→ローマ字変換（複数入力パターン対応） |
| `ScoreCalculator` | スコア計算ロジック（WPM、コンボ等） |
| `QuestionProvider` | 出題データの取得・シャッフル |

## 8. 非機能要件

### 8.1 パフォーマンス
| 指標 | 目標値 |
|------|--------|
| 初回ロード時間 | 3秒以内（LTE環境） |
| ゲーム内フレームレート | 60fps維持 |
| アセット最適化 | 画像圧縮、コードminify、Tree Shaking |

### 8.2 セキュリティ
- XSS対策（ユーザー入力のサニタイズ・エスケープ）
- CSP（Content Security Policy）ヘッダー設定
- 外部リソースの整合性チェック（SRI: Subresource Integrity）
- お問い合わせはGoogle Forms外部委託でサーバーサイドリスクを回避

### 8.3 アクセシビリティ
- セマンティックHTML
- キーボードナビゲーション対応
- 適切なコントラスト比（WCAG 2.1 AA準拠目標）

### 8.4 SEO
- メタタグ設定（title, description）
- OGP（Open Graph Protocol）設定
- 構造化データ（JSON-LD）

### 8.5 保守性
- DDD戦略設計に基づくモジュール分割
- コンテキスト間の疎結合
- 新規ゲーム追加が容易なプラグイン的構造
- 教育利用を考慮したコード可読性の維持

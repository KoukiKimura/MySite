## 実装ロードマップ

### Phase A: 設計確定
- ゲームルール、残機、スコア、コンボ仕様を確定する
- ステージ `1-1` から `1-5` の配置パターンを決める
- 画面遷移と結果画面の表示要素を確定する

### Phase B: ドメイン実装
- GameSession を実装する
- StageState を実装する
- ScoreCalculator を実装する
- セッション遷移、ライフ減算、ステージ進行、コンボ更新をテストする

### Phase C: プレゼンテーション実装
- Canvas 基盤を実装する
- マウス入力追従のバー制御を実装する
- ボール移動、衝突判定、描画ループを実装する
- オーバーレイ UI を実装する

### Phase D: 統合
- ルーターへページを追加する
- ポータルのゲーム一覧へ追加する
- サムネイルと説明ページを追加する
- `destroy()` でイベントや `requestAnimationFrame` を解放する

### Phase E: 品質確認
- ドメイン・アプリケーションテストを追加する
- `npm test` と `npm run build` を通す
- レビュー結果を設計書へ反映する

## リスクと対策

| リスク | 内容 | 対策 |
|--------|------|------|
| 衝突抜け | 高速移動時にボールがブロックをすり抜ける | 1フレームを複数サブステップに分割する |
| コンボ誤判定 | バー接触前後で連続破壊判定が崩れる | コンボ更新を GameSession に集約する |
| 画面破棄漏れ | ルート移動後も RAF やイベントが残る | `destroy()` で入力・描画・DOM を明示解放する |
| ステージ難度偏り | 1-1〜1-5 の差が弱い | 配置密度、ボール速度、バー幅で段階差を付ける |

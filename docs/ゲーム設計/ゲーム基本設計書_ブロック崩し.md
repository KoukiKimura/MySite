## 1. ゲーム概要

### 1.1 コンセプト
かわいいペンギンキャラクターをモチーフにしたネオン調ブロック崩しゲーム。パドルを操作してボールを反射させ、フィールド上のブロックを全て破壊してステージクリアを目指す。パワーアップアイテムで状況が変化するアーケードライクな爽快感が特徴。

### 1.2 対象ユーザー
- 手軽にゲームを楽しみたい全年齢のユーザー
- ブラウザゲームを楽しみたい小中学生〜大人
- タイピングゲームと並んでサイトに訪れるゲームユーザー

### 1.3 基本仕様

| 項目 | 内容 |
|------|------|
| ジャンル | ブロック崩し（アーケード） |
| 描画方式 | HTML5 Canvas |
| 入力方式 | マウス操作（PC推奨）、タッチ操作（モバイル）、キーボード（←→キー） |
| プレイ人数 | 1人 |
| 推奨環境 | PC / スマートフォン（タッチ対応） |

#### モバイル対応方針
- タッチ操作（指でパドルをスライド）に**完全対応**する
- ゲーム説明ページにてPC / モバイル双方でプレイ可能である旨を明示
- Canvasの描画サイズをデバイス幅に合わせてスケーリング（アスペクト比 3:2 固定）


#### GameSession 状態遷移

```
┌──────────┐  start()  ┌──────────┐
│  READY   │──────────►│ PLAYING  │
└──────────┘           └────┬─────┘
                            │
             ┌──────────────┼──────────────┐
             │ pause()      │ ボール全消滅  │ 全ブロック破壊
             ▼              ▼              ▼
       ┌──────────┐  ┌──────────┐  ┌──────────────┐
       │  PAUSED  │  │ LOST_LIFE│  │ STAGE_CLEAR  │
       └──────────┘  └──────────┘  └──────────────┘
             │              │              │
       resume()  残機>0→RDY  残機=0    最終ステージ    まだステージあり
             │        │       │              │              │
             ▼        ▼       ▼              ▼              ▼
       ┌──────────┐ READY GAME_OVER   ┌──────────┐   ┌──────────┐
       │ PLAYING  │                   │ FINISHED │   │  READY   │
       └──────────┘                   └──────────┘   └──────────┘
```

| 状態 | 説明 |
|------|------|
| `READY` | 初期状態またはミス後のリセット中。パドルにボール乗った状態 |
| `PLAYING` | プレイ中（ゲームループ稼働） |
| `PAUSED` | 一時停止中 |
| `LOST_LIFE` | ボールが全て画面外に消えた瞬間の遷移状態 |
| `STAGE_CLEAR` | 全破壊可能ブロック破壊後 |
| `GAME_OVER` | 残機が0になった |
| `FINISHED` | 全ステージクリア完了 |

### 2.4 ドメインサービス (Domain Services)

#### CollisionDetector — 衝突判定

```javascript
// domain/services/collision-detector.js
/**
 * ボール vs パドル、ボール vs ブロック、ボール vs 壁の衝突判定を行う。
 *
 * 判定方式: 円（ボール）と矩形（パドル/ブロック/壁）のAABB + 円矩形判定
 *
 * 主要メソッド:
 *   CollisionDetector.checkBallWall(ball, fieldWidth, fieldHeight) → { hitWall: boolean, normal: Vector2 }
 *   CollisionDetector.checkBallPaddle(ball, paddle) → { hit: boolean, normal: Vector2, reflectAngle: number }
 *   CollisionDetector.checkBallBlock(ball, block, blockRect) → { hit: boolean, normal: Vector2 }
 *   CollisionDetector.checkItemPaddle(item, paddle) → boolean
 */
```

パドル衝突時の反射角度計算:
- パドル中央への近さに応じて反射角を変化させる（中央 → 垂直反射、端 → 斜め反射）
- 最大反射角 75°（直角近辺での壁貼り付き防止）

#### PhysicsEngine — 物理演算

```javascript
// domain/services/physics-engine.js
/**
 * ボールの移動と反射を計算する。
 *
 * 主要メソッド:
 *   PhysicsEngine.updateBall(ball, delta) → Ball  移動後のボール（不変オブジェクト）
 *   PhysicsEngine.reflectBall(ball, normal) → Ball  法線に基づく反射
 *   PhysicsEngine.applySpeedMultiplier(ball, multiplier) → Ball  速度スケール変更
 */
```

#### ScoreCalculator — スコア計算

```javascript
// domain/services/score-calculator.js
/**
 * スコア計算ルール:
 *   ブロック破壊点 = BlockType.points × コンボ倍率
 *   コンボ倍率: combo 1〜4 → ×1.0 / 5〜9 → ×1.5 / 10〜19 → ×2.0 / 20〜 → ×3.0
 *   パワーアップ獲得: +50pt （固定）
 *   ステージクリアボーナス: 残機数 × 200pt
 *
 * 主要メソッド:
 *   ScoreCalculator.calcBlockScore(block, combo) → number
 *   ScoreCalculator.calcClearBonus(lives) → number
 */
```

#### StageBuilder — ステージ構築

```javascript
// domain/services/stage-builder.js
/**
 * ステージJSONデータからStageエンティティを構築する。
 *
 * 主要メソッド:
 *   StageBuilder.build(stageData) → Stage
 *
 * stageData は infrastructure/data/ のJSONと同形式。
 */
```

## 3. アプリケーション層 — ユースケース

### 3.1 ユースケース一覧

| ID | 名前 | 概要 |
|----|------|------|
| UC-1 | StartGame | ゲームセッション初期化、ステージ読み込み、カウントダウン開始 |
| UC-2 | UpdateFrame | 1フレーム分の物理演算・衝突処理・状態更新 |
| UC-3 | MovePaddle | 入力に基づくパドル位置更新 |
| UC-4 | ApplyPowerUp | パドルがアイテムを取得したときの効果適用 |
| UC-5 | LoseLife | ボール消滅後の残機減少・再セット |
| UC-6 | ClearStage | ステージクリア後の次ステージ遷移 |
| UC-7 | FinishGame | 全ステージクリア時のスコア確定 |

### 3.2 UC-2: UpdateFrame の処理フロー

```
UpdateFrame(deltaTime)
│
├── 各ボールに対して:
│   ├── PhysicsEngine.updateBall() — 位置更新
│   ├── CollisionDetector.checkBallWall() → 反射
│   ├── CollisionDetector.checkBallPaddle() → 反射
│   └── CollisionDetector.checkBallBlock() → ブロック毎
│       └── ヒット時: Block.hp減算 → 破壊判定
│           ├── 破壊: ScoreCalculator.calcBlockScore() → score加算
│           │         PowerUpType ブロック → アイテムスポーン
│           └── 全ブロック破壊: ClearStage ユースケース
│
├── 各落下アイテムに対して:
│   ├── 落下位置更新
│   └── CollisionDetector.checkItemPaddle() → ApplyPowerUp
│
├── ボールが全て LOST → LoseLife ユースケース
│
└── パワーアップ効果タイマー更新（duration 切れたら効果解除）
```

## 5. インフラ層

### 5.1 ステージJSONスキーマ

ステージデータは `infrastructure/data/stages-{difficulty}.json` で管理する。

```json
{
  "difficulty": "normal",
  "stages": [
    {
      "stageNumber": 1,
      "rows": 6,
      "cols": 11,
      "grid": [
        ["N","N","N","N","N","N","N","N","N","N","N"],
        ["N","H","N","N","P","N","N","N","H","N","N"],
        ["N","N","N","N","N","N","N","N","N","N","N"],
        ["I","N","N","N","N","N","N","N","N","N","I"],
        ["N","N","H","N","P","N","N","H","N","N","N"],
        ["N","N","N","N","N","N","N","N","N","N","N"]
      ]
    }
  ]
}
```

| グリッド値 | ブロック種別 |
|-----------|------------|
| `"N"` | 通常（NORMAL） |
| `"H"` | 硬い（HARD） |
| `"I"` | 無敵（INVINCIBLE） |
| `"P"` | パワーアップ（POWER_UP） |
| `""` or `null` | ブロックなし（空き） |
